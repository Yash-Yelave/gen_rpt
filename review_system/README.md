# AI Review System

The **AI Review System** is an independent, evidence-based report auditor designed to evaluate, score, and provide actionable feedback on reports generated by the `gen_rpt` engine (or any markdown/text report).

---

## 🚀 How to Execute

To run a review on a report, execute the `main.py` script from the project root directory and provide the path to your report file.

### Basic Usage
```bash
python review_system/main.py --report path/to/your/report.md
```

### Advanced Usage (Specify Model)
You can optionally specify a specific Groq model to use:
```bash
python review_system/main.py --report path/to/your/report.md --model llama-3.3-70b-versatile
```

### Outputs
The system will automatically generate a folder inside the root `review_outputs/` directory. 
- If your report is uniquely named (e.g., `test_report.md`), the output folder will be `review_outputs/test_report_review/`.
- If your report is generically named inside a project folder (e.g., `reports/Project_Alpha/report.md`), the output folder will be `review_outputs/Project_Alpha_review/`.

## 🧩 Components & Architecture

The review system is modularly designed into distinct components to separate concerns (extraction, reviewing, scoring, and output generation).

### 1. `main.py` (Entry Point)
The command-line interface and orchestrator that ties all the components together. It handles path resolution, folder naming, logging, and kicking off the review pipeline.

### 2. `extractors/`
Responsible for reading and parsing the input reports.
- **`report_loader.py`**: Safely loads the file content and detects the title.
- **`section_parser.py`**: Chunks the report into logical sections so the AI can evaluate it effectively without overwhelming the context window.
- **`claim_extractor.py`**: Extracts factual claims from the text for independent verification.

### 3. `reviewers/`
The core AI logic that interacts with the Groq API to perform the actual audit.
- **`groq_review_engine.py`**: Handles API calls to Groq.
- **`review_orchestrator.py`**: Manages the multi-step review pipeline (overall assessment, tone check, structural analysis).
- **`review_builder.py`**: Assembles the raw AI output into a structured `ReviewData` object.

### 4. `scoring/`
Responsible for grading the report.
- Evaluates the report based on multiple criteria (e.g., Strategic Alignment, Clarity, Evidence) to generate an objective numerical score and a letter grade.

### 5. `outputs/`
Responsible for rendering the final `ReviewData` into human-readable and machine-readable artifacts.
- **`json_writer.py`**: Dumps structured data (scores, claims, tasks) for automated pipelines.
- **`markdown_writer.py`**: Generates a pretty `.md` summary of the review.
- **`html_writer.py`**: Generates a visually appealing `.html` dashboard of the review results.

### 6. `config/`
Contains configuration files like `review_config.py` and `prompts.py` which define the default AI models, system prompts, and behavior rules.

### 7. `utils/`
Helper functions for logging, directory management, and file path resolution.

### 8. `logs/`
Contains log files (`review_run.log`, `review_generation.log`) that track the execution steps and record any errors encountered during the review process.
